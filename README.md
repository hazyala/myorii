# 🐈 Myorii

> 메뉴바에 사는 작은 고양이 생산성 도우미

Myorii는 macOS 메뉴바 및 Windows 시스템 트레이에 상주하며, 사용자가 가장 자주 사용하는 생산성 기능을 창 이동 없이 즉시 사용할 수 있도록 돕는 로컬 LLM 기반 데스크톱 컴패니언 애플리케이션입니다.

---

# ✨ 프로젝트 소개

개발 작업이나 문서 작업 중에는 다음과 같은 불편함이 자주 발생합니다.

* 변수명, 함수명, 파일명을 정하기 위해 번역기나 AI를 반복적으로 열어야 함
* 간단한 질문을 하기 위해 새로운 브라우저 탭을 열어야 함
* 메모를 위해 별도 앱을 실행해야 함
* 여러 생산성 도구를 오가며 작업 흐름이 끊김

Myorii는 이러한 문제를 해결하기 위해 메뉴바에 항상 상주하며, 작은 팝업 창 하나로 네이밍, 채팅, 메모 기능을 제공합니다.

# ▶ 실행 방법

## 가상환경 생성

```bash
python3 -m venv .venv
```

## 활성화

```bash
source .venv/bin/activate
```

## 패키지 설치

```bash
pip install -r requirements.txt
```

## Ollama 실행

```bash
ollama serve
```

## 모델 다운로드

```bash
ollama pull qwen3-vl:4b
```

## 실행

개발 중에는 아래 명령으로 실행한다.

```bash
python main.py
```

현재 구현 기준으로 Myorii 메뉴바 아이콘을 클릭하면 메인 창이 바로 토글된다.

## macOS 앱 번들 빌드

```bash
PYINSTALLER_CONFIG_DIR=/tmp/myorii_pyinstaller .venv/bin/pyinstaller packaging/macos/Myorii.spec --noconfirm --clean
open -n dist/Myorii.app
```

Myorii는 중복 메뉴바 아이콘 생성을 막기 위해 단일 실행 잠금을 사용한다. 새로 빌드한 앱을 확인할 때는 기존 Myorii 프로세스를 먼저 종료한 뒤 실행한다.

* 첫 번째 클릭: 아이콘 위치 근처에 메인 창 표시
* 두 번째 클릭: 메인 창 닫기
* Header의 `×` 버튼: 메인 창 닫기
* 이미 실행 중인 상태에서 다시 실행하면 새 프로세스는 바로 종료되어 메뉴바 아이콘이 중복 생성되지 않는다.

메뉴바 아이콘은 현재 36px 기준으로 렌더링한다.

## 현재 UI 구현 범위

현재 메인 창은 채팅, 할일, 메모 탭이 포함된 PyQt6 팝오버 화면이다.

* 채팅, 할일, 메모 탭은 하나만 선택된다.
* 선택된 탭에 맞춰 중앙 컨텐츠 영역이 전환된다.
* 채팅 탭은 Ollama `qwen3-vl:4b` 모델과 연결되어 사용자 메시지와 AI 응답을 표시한다.
* 채팅 요청은 `InstantRouter`, `IntentRouter`, `ModelRouter`, `PromptProfileResolver`, `ResponseFormatter`를 거쳐 처리한다.
* 짧은 단어 번역과 식사/커피 추천 일부는 Ollama 호출 없이 즉시 응답한다.
* 이미지가 없는 텍스트 요청은 설치된 fast text 모델을 우선 사용하고, 이미지 요청은 vision 모델을 사용한다.
* 응답은 워커 스레드에서 토큰 단위로 수신하고, UI에는 스트리밍으로 반영된다.
* Assistant 응답은 완료 시점에 Markdown으로 렌더링한다.
* 코드블록, 인라인 코드, 파일명/함수명/변수명/명령어 후보는 복사 가능한 블록으로 표시하고, 클릭 시 클립보드에 복사되며 `복사됨` 토스트를 표시한다.
* 여러 후보는 후보별 코드블록으로 분리하고, 하나의 코드 스니펫이나 셸 스크립트는 하나의 코드블록으로 유지한다.
* 응답 대기 중에는 빨간 발바닥 3개가 순차로 튀는 인디케이터를 표시하고, 첫 토큰 수신 시 텍스트로 전환한다.
* 현재 세션의 대화 맥락은 앱 실행 중 `ChatService`가 유지한다.
* Header에는 인터넷 연결 상태에 따라 `온라인` 또는 `오프라인`이 앱 이름 옆에 표시된다.
* 채팅 영역은 메시지 목록만 스크롤되도록 독립 스크롤 영역으로 구성한다.
* 채팅 입력 영역 상단에는 채팅 기록 버튼과 대화 기록 저장 스위치가 포함된다. 현재는 토글 상태만 보관한다.
* 입력창 내부 좌측의 `+` 버튼으로 로컬 파일을 선택해 첨부할 수 있다.
* 채팅 입력부에 파일을 드래그 앤 드롭하면 첨부 파일로 추가된다.
* 첨부 파일은 한 번에 1개만 추가할 수 있으며, 파일 크기는 최대 5MB까지 허용한다.
* 첨부 파일은 입력부 상단에 미리보기 카드로 표시되며, 긴 파일명은 카드 폭에 맞춰 말줄임 처리한다.
* 전송된 사용자 메시지의 첨부 파일은 메시지 텍스트와 분리해 사용자 버블 상단에 별도 미리보기 카드로 표시하고, 카드가 길어지면 줄바꿈해 하단으로 이어진다.
* 이미지 첨부 파일은 `ImageHandler`가 base64로 변환한 뒤 Ollama 요청의 `messages[].images`에 포함해 vision 모델에 전달한다.
* `TextHandler`는 `txt`, `md`, `json`, `yaml`, `yml` 첨부의 본문 일부를 UTF-8 기준으로 추출하고, JSON은 읽기 쉬운 들여쓰기 형식으로 정리한다.
* `CsvHandler`는 `csv`, `tsv` 첨부의 컬럼명과 샘플 행을 요약한다.
* `PdfHandler`는 `pdf` 첨부에서 앞쪽 일부 페이지의 텍스트를 추출해 첨부파일 참고 내용으로 전달한다. 스캔 PDF, OCR, 표 구조 보존, 이미지/차트 분석은 지원하지 않는다.
* `DocxHandler`는 `docx` 첨부의 문단과 표 안의 텍스트 일부를 첨부파일 참고 내용으로 전달한다. 서식, 이미지, 주석, 변경 추적, 매크로는 분석하지 않는다.
* `XlsxHandler`는 `xlsx` 첨부의 시트명, 컬럼명, 샘플 행을 첨부파일 참고 내용으로 전달한다. 전체 행 분석, 수식 재계산, 피벗/차트/서식 분석은 지원하지 않는다.
* `PptxHandler`는 `pptx` 첨부의 슬라이드별 텍스트를 첨부파일 참고 내용으로 전달한다. PPT 제작, 디자인 생성, 이미지/차트/표 구조 분석, 발표자 노트, 수치 계산은 지원하지 않는다.
* `ChatService`는 텍스트, CSV/TSV, PDF, DOCX, XLSX, PPTX 첨부파일 참고 내용을 user message 본문과 metadata에 추가해 모델 요청에 포함한다.
* 내용이 많거나 복잡한 첨부파일은 일부 내용만 참고할 수 있음을 사용자용 문구로 안내한다.
* 지원 파일 형식은 이미지(`png`, `jpg`, `jpeg`, `gif`, `bmp`), 텍스트(`txt`, `md`, `json`, `yaml`, `yml`), 표 텍스트(`csv`, `tsv`), 문서/오피스(`pdf`, `docx`, `xlsx`, `pptx`)로 제한한다.
* 지원하지 않는 파일은 Assistant 메시지로 오류를 표시한다.
* 입력창은 `Enter`로 전송하고 `Shift+Enter`로 줄바꿈한다.
* 긴 입력 텍스트는 입력창 폭에 맞춰 자동 줄바꿈된다.
* Header의 설정 아이콘을 클릭하면 설정 화면으로 전환되고, `×` 버튼을 클릭하면 메인 창을 숨긴다.
* 할일 탭은 로컬 SQLite 저장, 완료 체크, 드래그 재정렬, 새 할 일 추가를 지원한다.
* 메모 탭은 로컬 SQLite 저장, 목록/편집 화면 전환, 자동 저장, 삭제, 드래그 재정렬을 지원한다.
* 새로운 메모를 클릭하면 바로 전체 메모 작성 화면으로 전환된다.
* 메모 입력은 Markdown 서식을 지원하며 제목, 목록, 체크박스, 인용, 인라인 코드, 코드 블럭을 편집 중 즉시 시각화한다.
* 메모 코드 블럭 안에서 `Enter`는 블럭을 닫고 밖으로 이동하며, `Shift+Enter`는 블럭 내부 줄바꿈을 유지한다.
* 메모 목록 카드 본문은 최대 2줄로 말줄임 처리되어 날짜 영역과 겹치지 않는다.

현재 설정 화면은 환경설정 UI의 기본 구조와 일부 실제 동작만 구현되어 있다.

* 일반 섹션에는 시작 시 Myorii 열기 스위치, 라이트/다크 테마 토글, 한국어/영어 언어 토글을 배치한다.
* 로컬 모델 섹션에는 설치된 Ollama 모델 드롭다운과 모델 관리 버튼을 배치한다.
* 기본 모델 `qwen3-vl:4b`는 설치 모델 목록과 무관하게 드롭다운 첫 항목으로 표시한다.
* 설치 모델 목록은 앱 시작 후 워커 스레드에서 비동기로 로드해 시작 지연을 막는다.
* 연동 섹션에는 Notion 연동 버튼을 배치한다.
* 정보 섹션에는 임시 버전과 도움말, 피드백 버튼을 배치한다.
* 앱 종료 버튼은 실제로 애플리케이션을 종료한다.
* 모델 관리, 외부 API 연동, 도움말, 피드백의 실제 기능은 추후 구현한다.

설정 화면의 전체 폰트 크기, 여백, 배치, 버튼 모양, 아이콘과 글자 간격은 추후 디자인 튜닝 대상이다.

## 실행 종료

설정 화면의 앱 종료 버튼을 누르면 Myorii를 종료한다.

개발 중 강제 종료가 필요하면 프로세스를 직접 종료할 수 있다.

```bash
pkill -f Myorii
```

터미널에서 `python main.py`로 실행한 경우에는 `Ctrl+C`로 종료할 수 있다.
